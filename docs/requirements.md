# ミニマムタックス M&Aシミュレーションアプリ 要件定義書

**作成日**: 2026年3月13日  
**バージョン**: 1.1  
**ステータス**: 主要要件確定

---

## 1. プロジェクト概要

### 1.1 目的
2026年度（令和8年度）税制改正による「極めて高い水準の所得に対する負担の適正化措置」（通称：ミニマムタックス）の見直しが、M&Aにおけるオーナー社長の**株式譲渡後の手取り額**に与える影響を可視化するシミュレーションアプリを構築する。

### 1.2 背景
令和8年度税制改正大綱（令和7年12月26日閣議決定）により、ミニマムタックスの主要パラメータが以下のとおり大幅に強化される。

| 項目 | 現行（〜2026年分） | 改正後（2027年分〜） |
|------|-------------------|---------------------|
| 特別控除額 | **3億3,000万円** | **1億6,500万円** |
| 税率 | **22.5%** | **30%** |
| 発動目安（株式譲渡のみの場合） | 約10億円超 | **約3.4億円超** |

この改正により、従来は「超富裕層向けのニッチな制度」だったミニマムタックスが、**中堅企業のM&Aでも影響が出る制度**へと変わった。M&Aアドバイザーがクライアント（オーナー社長）に対し、売却タイミングや手取り額への影響を正確かつ直感的に説明できるツールが必要となる。

### 1.3 利用者・利用シーン
- **主な利用者**: M&Aアドバイザー（操作者）
- **最終閲覧者**: クライアント（オーナー社長）
- **利用シーン**: 
  - 初期提案時の税務インパクト説明
  - 売却タイミング（2026年内 vs 2027年以降）の意思決定支援
  - 一括売却 vs 分割売却の税負担比較
  - 複数の譲渡価額シナリオでの手取り額比較

### 1.4 プラットフォーム
- Webアプリケーション（ブラウザで動作）
- レスポンシブ対応（PCでの提示を主想定、タブレットでも閲覧可能）
- サーバーサイド不要（クライアントサイドで完結する計算ロジック）

## 2. 機能要件

### 2.1 入力パラメータ

#### 2.1.1 必須入力項目

| # | 項目名 | 入力形式 | 単位 | デフォルト値 | 備考 |
|---|--------|----------|------|-------------|------|
| 1 | 株式譲渡価額（売却価格） | 数値入力 | 万円 | なし | スライダーとの併用可 |
| 2 | 株式取得価額（簿価） | 数値入力 | 万円 | 0 | 創業オーナーの場合ほぼゼロ |

> **設計判断**: 「譲渡所得」の直接入力ではなく「譲渡価額」と「取得価額」を分離入力とする。理由は以下のとおり。
> - オーナー社長は「いくらで売れるか」と「いくらで取得したか」で考える（譲渡益という抽象概念より直感的）
> - 創業オーナーは取得価額＝資本金（数百万円〜数千万円）が多く、デフォルト0円で概算可能
> - M&Aアドバイザーが提示するバリュエーションレンジとの整合性が取りやすい

#### 2.1.2 任意入力項目（詳細モード）

| # | 項目名 | 入力形式 | 単位 | デフォルト値 | 備考 |
|---|--------|----------|------|-------------|------|
| 3 | 給与所得（年収） | 数値入力 | 万円 | 0 | 役員報酬等 |
| 4 | 不動産所得 | 数値入力 | 万円 | 0 | 賃貸収入等 |
| 5 | その他の総合課税所得 | 数値入力 | 万円 | 0 | 事業所得等 |
| 6 | 上場株式の譲渡所得 | 数値入力 | 万円 | 0 | 申告分離課税 |
| 7 | 上場株式の配当所得 | 数値入力 | 万円 | 0 | 申告分離選択時 |
| 8 | 所得控除合計 | 数値入力 | 万円 | 200 | 基礎控除＋社会保険料控除等の概算 |

#### 2.1.3 入力UIの要件
- **シンプルモード / 詳細モードの切替**: 初期表示はシンプルモード（項目#1, #2のみ）。詳細モードで#3〜#8を展開。
- **スライダー**: 株式譲渡価額にはスライダーを設置（範囲: 0〜100億円、0.1億円刻み）。数値入力との同期。
- **プリセットボタン**: 代表的な譲渡金額（3億円 / 5億円 / 10億円 / 20億円 / 50億円）をワンクリックで入力可能に。

### 2.2 計算ロジック

#### 2.2.1 基本計算フロー

```
Step 1: 株式譲渡所得 = 株式譲渡価額 − 株式取得価額
Step 2: 基準所得金額 = 株式譲渡所得 + 総合課税所得 + 上場株式譲渡所得 + 上場株式配当所得
       （※源泉分離課税所得、NISA口座の所得は含まない）
Step 3: 通常の所得税額（基準所得税額）を計算
Step 4: ミニマムタックス判定・追加税額計算
Step 5: 住民税の計算
Step 6: 復興特別所得税の計算
Step 7: 手取り額 = 株式譲渡価額 − 税額合計
```

#### 2.2.2 通常の所得税額（基準所得税額）の計算

**（A）総合課税部分**

給与所得については給与所得控除を適用した上で、以下の累進税率を適用する。

| 課税所得金額 | 税率 | 控除額 |
|-------------|------|--------|
| 〜195万円 | 5% | 0円 |
| 195万円超〜330万円 | 10% | 97,500円 |
| 330万円超〜695万円 | 20% | 427,500円 |
| 695万円超〜900万円 | 23% | 636,000円 |
| 900万円超〜1,800万円 | 33% | 1,536,000円 |
| 1,800万円超〜4,000万円 | 40% | 2,796,000円 |
| 4,000万円超 | 45% | 4,796,000円 |

> **給与所得控除**（概算テーブル）: 
> - 850万円超: 上限195万円（令和8年分〜の改正後も同様）
> - 詳細は累進テーブルに準ずる

**（B）分離課税部分（株式譲渡所得・上場株式配当）**

所得税率: **15%**（一律）

**（C）基準所得税額**

```
基準所得税額 = (A) + (B)
```

#### 2.2.3 ミニマムタックスの計算

**改正前（2025年〜2026年分）**:
```
追加税額 = max(0, (基準所得金額 − 33,000万円) × 22.5% − 基準所得税額)
```

**改正後（2027年分〜）**:
```
追加税額 = max(0, (基準所得金額 − 16,500万円) × 30% − 基準所得税額)
```

#### 2.2.4 住民税の計算

ミニマムタックスは**所得税（国税）のみに適用**され、住民税には影響しない。

- 総合課税部分: 課税所得 × **10%**（均等割は簡便化のため省略）
- 分離課税部分（株式譲渡所得）: 譲渡所得 × **5%**

#### 2.2.5 復興特別所得税

```
復興特別所得税 = (通常の所得税額 + ミニマムタックス追加税額) × 2.1%
```

> **注記**: 令和8年度税制改正で防衛特別所得税（仮称）の創設と復興特別所得税の税率引下げが決定されているが、所得税額に対する付加税率は合計 **2.1%** が当面維持される見込み（令和9年1月施行）。シミュレーション上は2.1%を適用する。

#### 2.2.6 税額合計と手取り額

```
所得税合計 = 通常の所得税額 + ミニマムタックス追加税額
復興特別所得税 = 所得税合計 × 2.1%
住民税合計 = 住民税（総合課税分 + 分離課税分）
-----------------------------------------------
税額合計 = 所得税合計 + 復興特別所得税 + 住民税合計
```

**手取り額の表示モード（トグル切替）**:

| モード | 定義 | 主な利用場面 |
|--------|------|-------------|
| **M&A手取りモード**（デフォルト） | 手取り額 = 株式譲渡価額 − 株式譲渡に帰属する税額 | 「会社を売ったらいくら残るか」を端的に示す場面 |
| **年間総合モード** | 手取り額 = 全所得合計 − 税額合計（全所得に対する税額） | 他の所得も含めた年間の税負担全体を俯瞰する場面 |

> **設計意図**: オーナー社長が最も関心を持つのは「売却代金からいくら手元に残るか」であるため、M&A手取りモードをデフォルトとする。一方、役員報酬や不動産所得がある場合、ミニマムタックスがそれらの税負担にも波及的に影響する（基準所得金額が押し上げられる）ため、年間総合モードも必要。トグルで切替可能とし、切替時にラベルと計算結果が連動して変わる。

> **M&A手取りモードにおける税額按分ロジック**: ミニマムタックスの追加税額は所得全体に対して算出されるため、株式譲渡に帰属する税額は以下の按分で算出する。
> ```
> 株式譲渡帰属の追加税額 = 追加税額 × (株式譲渡所得 ÷ 基準所得金額)
> ```

### 2.4 分割売却シミュレーション機能

M&Aの売却タイミングに関する最も重要な意思決定支援機能。一括売却と2年分割売却の税負担を比較する。

#### 2.4.1 機能概要

オーナー社長が保有株式を**2026年中に全量売却（改正前）**するか、**一部を2026年・残りを2027年に分割売却**するかを比較検討するための機能。

> **背景**: 所得税の特例により、株式譲渡契約の締結が2026年中であれば2026年分の所得として申告可能。一方、M&Aにおいては段階的な株式取得（例: 51%→100%のステップアップ）や、アーンアウト条項による段階的支払いも一般的であり、分割売却の検討ニーズは高い。

#### 2.4.2 入力項目（分割売却モード）

| # | 項目名 | 入力形式 | デフォルト値 | 備考 |
|---|--------|----------|-------------|------|
| 1 | 株式譲渡価額（総額） | 数値入力 | メイン画面から引継ぎ | |
| 2 | 株式取得価額（総額） | 数値入力 | メイン画面から引継ぎ | |
| 3 | 1年目（2026年）の売却割合 | スライダー | 50% | 10%刻み |
| 4 | 各年の他の所得 | 数値入力 | メインの詳細モードと連動 | 各年同額とする簡略化 |

> **取得価額の按分**: 売却割合に応じて取得価額も同比率で按分する（簿価は均一と仮定）。

#### 2.4.3 比較出力

3つのシナリオを横並びで比較表示する。

| 比較項目 | シナリオA | シナリオB | シナリオC |
|---------|----------|----------|----------|
| **内容** | 2026年に全量売却 | 2027年に全量売却 | 2026年にX%、2027年に残り |
| 適用制度 | 改正前 | 改正後 | 1年目: 改正前 / 2年目: 改正後 |
| 1年目 税額 | ○ | — | ○ |
| 2年目 税額 | — | ○ | ○ |
| **税額合計（2年間）** | **○** | **○** | **○** |
| **手取り合計（2年間）** | **○** | **○** | **○** |
| **シナリオAとの差額** | — | **○** | **○** |

#### 2.4.4 分割売却の最適比率チャート

- X軸: 1年目（2026年）の売却割合（0%〜100%）
- Y軸: 2年間合計の税額
- **税額が最小となる分割比率**をハイライト表示
- ただし、実務上はM&Aの取引構造（支配権移転のタイミング等）に制約されるため、「税務上の最適解が実務上実行可能とは限らない」旨の注記を付す

#### 2.4.5 計算上の前提・制約

- 各年の所得は独立して計算（損益通算・繰越控除は未対応）
- 2年目（2027年）の他の所得は1年目と同額と仮定（簡略化）
- 分割売却の場合、各年の基準所得金額は各年の譲渡所得＋他の所得で計算
- M&Aの実行可能性（SPA締結時期、クロージング条件等）はシミュレーション対象外

### 2.5 補助機能

| # | 機能 | 説明 |
|---|------|------|
| 1 | PDF出力 | 入力条件・計算結果・チャートをPDFとして出力。クライアントへの説明資料として利用 |
| 2 | 入力条件の保存/復元 | URLパラメータまたはローカルストレージによる条件の保持 |
| 3 | 免責条項表示 | 「本シミュレーションは概算であり、正確な税額は税理士にご確認ください」等 |
| 4 | 制度解説パネル | ミニマムタックスの仕組みをオーナー社長向けに平易に解説するサイドパネル |

## 3. 非機能要件

### 3.1 パフォーマンス
- スライダー操作に対してリアルタイム（100ms以内）で計算結果を反映
- 初期ロード: 3秒以内

### 3.2 セキュリティ
- サーバーサイド通信なし（すべてクライアントサイドで完結）
- 入力データは外部に送信しない（クライアントの機微情報保護）

### 3.3 ブラウザ互換性
- Chrome、Safari、Edge（最新版）
- iPadのSafariでも正常表示

### 3.4 アクセシビリティ
- 金額の桁区切り表示（例: 1,650,000,000円 → 16億5,000万円）
- カラーユニバーサルデザイン配慮（色覚異常対応）

## 4. 計算精度に関する注意事項・制約

### 4.1 簡略化する項目
本シミュレーションは**M&Aの意思決定支援**が目的であり、確定申告レベルの精緻さは求めない。以下を簡略化する。

| 項目 | 対応 |
|------|------|
| 給与所得控除 | 概算テーブル適用（上限195万円） |
| 所得控除 | 一括入力（デフォルト200万円） |
| 配偶者控除・扶養控除の逓減 | 未対応 |
| 損益通算・繰越控除 | 未対応 |
| エンジェル税制の非課税枠 | 未対応（必要なら将来バージョンで追加） |
| NISA口座の非課税枠 | 基準所得金額から除外済み（入力項目に含めない） |
| 均等割（住民税） | 省略 |

### 4.2 制度上の留意点（免責事項に記載すべき事項）
- 基準所得金額には「合計所得金額」に準ずるが、厳密には租税特別措置法第41条の19第2項に定める所得が対象であり、通常の合計所得金額とは異なる
- 源泉分離課税所得（利子所得等）は基準所得金額に含まれない
- 株式譲渡契約の締結が2026年中であれば、引渡しが2027年でも2026年分として申告可能な特例がある
- 本シミュレーションは2026年3月時点の税制改正大綱に基づくものであり、国会での法案成立・施行細則の確定により変更の可能性がある

## 5. UI/UXデザイン方針

### 5.1 全体構成

```
┌─────────────────────────────────────────────────┐
│  ヘッダー: 会社ロゴ + アプリタイトル + 制度解説   │
├─────────────────────────────────────────────────┤
│  タブ: [一括売却シミュレーション] [分割売却比較]   │
├───────────────┬─────────────────────────────────┤
│               │                                 │
│  入力パネル    │   結果表示エリア                 │
│  （左サイド）  │   ├ メイン比較カード             │
│               │   ├ 手取り額モード切替トグル       │
│  ・シンプル    │   ├ チャートエリア               │
│  ・詳細切替    │   └ 感応度テーブル               │
│               │                                 │
├───────────────┴─────────────────────────────────┤
│  免責条項フッター + © 会社名                      │
└─────────────────────────────────────────────────┘
```

### 5.2 ブランディング
- **ヘッダー**: 会社ロゴ（SVGまたはPNG）を左上に配置。ロゴ画像はデプロイ時に差し替え可能な設計とする
- **フッター**: 「© [会社名] All rights reserved.」を表示
- **PDF出力時**: ヘッダーにロゴ、フッターに免責条項＋会社名を記載
- **カラースキーム**: 会社のブランドカラーに合わせてCSS変数で一括変更可能な設計

### 5.3 デザイントーン
- **プロフェッショナル**: クライアント提示用のため、落ち着いた配色（ダークネイビー、白、アクセントにゴールド）
- **信頼感**: 金融・税務アドバイザリーに適したトーン
- **データ重視**: 装飾を最小限に抑え、数字が主役

### 5.4 金額表示ルール
- 1億円未満: 「○,○○○万円」
- 1億円以上: 「○○億○,○○○万円」
- 税率: 小数点第2位まで（例: 20.32%）
- 増税額: 赤字＋▲マーク

## 8. 検証用テストケース

### テストケース1: 株式譲渡のみ・5億円
- 譲渡価額: 5億円、取得価額: 0円
- 改正前: ミニマムタックス発動なし → 手取り約3億9,842万円
- 改正後: ミニマムタックス発動 → 手取り減少

### テストケース2: 株式譲渡のみ・10億円
- 譲渡価額: 10億円、取得価額: 0円
- 改正前: ほぼ発動閾値付近
- 改正後: 大幅な追加税額発生

### テストケース3: 株式譲渡のみ・3億円
- 譲渡価額: 3億円、取得価額: 0円
- 改正前: 発動なし
- 改正後: 発動なし（閾値以下）
- → 「影響なし」が正しく表示されることの確認

### テストケース4: 複合所得（給与3,000万円 + 株式譲渡10億円）
- 総合課税の所得税が基準所得税額に加算されることで、ミニマムタックスの追加税額が変動する挙動を確認

### テストケース5: 境界値
- 改正後の発動閾値付近（株式譲渡所得 3.3〜3.5億円あたり）で正しく判定されることを確認

### テストケース6: 分割売却（10億円を50:50で分割）
- 譲渡価額: 10億円（総額）、取得価額: 0円
- シナリオA: 2026年に全量売却（改正前適用）
- シナリオB: 2027年に全量売却（改正後適用）
- シナリオC: 2026年に5億円、2027年に5億円を売却
- → 各年のミニマムタックス発動有無と2年間合計の税額を正しく比較できることを確認
- → シナリオCでは1年目（5億円、改正前）はミニマムタックス発動なし、2年目（5億円、改正後）は発動あり、という非対称な結果が正しく表示されること

### テストケース7: 手取り額モード切替
- 給与3,000万円 + 株式譲渡10億円の入力で、M&A手取りモードと年間総合モードを切り替えた際に、表示金額が適切に変わることを確認
